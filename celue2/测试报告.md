# 策略2测试报告

**测试时间**：2026-06-24  
**测试目录**：`/home/p4/yws-covert/celue2`  
**测试目标**：验证策略2在不同链路时延和随机丢包率下能否恢复隐蔽数据。

## 1. 测试对象

策略2为高可靠 IP-ID 存储型隐蔽信道。当前版本使用：

1. IPv4 ID 标记 `covert_flag + strategy_id + seq_mod + encrypted_value`。
2. 块内 12 个数据半字节、3 个 XOR 冗余半字节、1 个块认证半字节。
3. 每个逻辑片段重复发送 3 次，提高随机丢包下的可恢复性。

## 2. 输入数据

| 项目 | 值 |
|---|---:|
| 输入文件 | `celue2/input_payload.bin` |
| 输入长度 | 64 字节 |
| 承载包数 | 576 个 |
| 发送方式 | h1 使用 `scapy-l2` 设置 IPv4 ID |
| 接收方式 | h2 sniff UDP 包并解析 IPv4 ID |

## 3. 测试结果

| 用例 | 配置时延(ms) | 配置丢包 | 收到包数 | 估计实际丢包 | 解码成功 | 数据一致 | 解码字节 |
|---|---:|---:|---:|---:|---|---|---:|
| delay5_loss0 | 5 | 0% | 576/576 | 0.0% | true | true | 64 |
| delay20_loss0 | 20 | 0% | 576/576 | 0.0% | true | true | 64 |
| delay50_loss0 | 50 | 0% | 576/576 | 0.0% | true | true | 64 |
| delay5_loss5 | 5 | 5% | 547/576 | 5.03% | true | true | 64 |
| delay5_loss10 | 5 | 10% | 523/576 | 9.2% | true | true | 64 |
| delay5_loss20 | 5 | 20% | 435/576 | 24.48% | true | true | 64 |
| delay5_loss30 | 5 | 30% | 393/576 | 31.77% | true | true | 64 |

## 4. 结论

本轮测试中，策略2在无丢包、5%、10%、20%、30% 随机丢包设置下均成功恢复 64 字节隐蔽数据，所有用例 `hidden_match=true`。

策略2的可靠性来自三次重复片段、XOR 冗余和块认证，代价是容量低于策略0/1。它适合后续由 INT 判断链路丢包率较高时启用。
