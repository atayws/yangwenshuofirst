# 策略2验证操作文档

## 1. 功能目标

策略2用于验证“抗丢包的 IP-ID 存储型隐蔽信道”。发送端 h1 通过 Scapy 二层发包显式设置 IPv4 Identification 字段，P4 交换机只负责转发，接收端 h2 抓包读取 IP-ID 并解码。

## 2. 编码格式

```text
bit15       covert_flag = 1
bits14-12   strategy_id = 2
bits11-8    seq_mod，块内片段序号 0~15
bits7-0     encrypted_value
```

`encrypted_value` 解密后为：

```text
高4 bit      block_mod，块号模16
低4 bit      data_nibble，数据/冗余/认证半字节
```

每块 16 个片段：0~11 为数据半字节，12~14 为 XOR 冗余半字节，15 为块认证半字节。当前每个片段默认重复发送 3 次，用于抵抗随机丢包。

## 3. 一键验证

```bash
cd /home/p4/yws-covert
sudo python3 celue2/run_strategy2_matrix.py --timeout 25
```

脚本会自动启动拓扑、下发流表、设置链路0的时延/丢包、发送策略2数据、接收并解码，最后清理 Mininet。

## 4. 结果文件

```text
celue2/input_payload.bin                 原始隐蔽数据
celue2/input_bits.txt                    原始 bit 串
celue2/results/summary.json              总结果
celue2/results/summary.csv               CSV 汇总
celue2/results/<case>/case_result.json   单用例结果
celue2/results/<case>/decoded_output.bin 解码输出
```

## 5. 单用例手动命令

如果已经在 `sudo bash run.sh` 的 Mininet CLI 中，可以执行：

```bash
h2 bash celue2/rx_strategy2.sh 50202 576 64 smoke &
h1 bash celue2/tx_strategy2.sh 50202 smoke
```

检查输出：

```bash
h2 cat celue2/results/smoke/receive_summary.json
h2 md5sum celue2/input_payload.bin
h2 md5sum celue2/results/smoke/decoded_output.bin
```

两个 MD5 一致说明隐蔽数据恢复成功。
