## 测试配置
- 拓扑：h1 -- s1 == 三条链路 == s2 -- h2
- 业务流：h2 作为 iperf client，h1 作为 iperf server，持续发送 UDP 业务流。
- INT 采样间隔：10.0 ms。
- 本轮快速复测：轮询多路径 15 s，UDP 负载 2 Mbit/s，业务包长 1200 字节。
- INT 方式：s2 在采样命中的真实业务包中 inline 插入 INT，s1 补齐两跳 probe_data 后剥离 INT 并恢复业务包，同时本地生成 UDP/50100 报告给 h1。
- MTU：终端侧 1500，s1-s2 三条交换机间链路 1600。

## 结论
- 业务连续性：18/18 条记录对应的 case 中 iperf 正常完成。
- INT 可观测性：18/18 条链路记录有 INT 指标样本。
- 时延：实测值整体等于链路配置时延加上 BMv2/Mininet 处理开销，适合用来比较三条路径优劣。
- 丢包：当前按每条路径 INT 序号缺口估算，能反映链路丢包趋势；短时间测试下会有采样随机误差。
- 带宽：报告 JSON 中保留了 `bw_utilization`，阶段1建议作为相对负载指标，不作为精确带宽计量。
