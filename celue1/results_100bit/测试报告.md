# 策略1：高容量排序时序 约100bit多条件测试

- 测试数据长度：104 bit（13字节，接近100bit）。
- 输入数据：`10111101101010011110011101001110110101010000011100101000010001110101101011011010101110100001100101101101`
- 承载流：h1 -> h2 低速可控 UDP 业务流，固定 path0。
- 不使用默认 TCP iperf 打满链路，避免排队时延改变策略阈值。
- 如果后续要加背景业务流，应使用限速背景流，例如 `iperf -u -b 1M`。
